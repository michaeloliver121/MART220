Process: 

I think this was one of those weird projects where I felt like I started without really knowing much of anything, so when I continued to add more to it, it kind of felt a little weird since the baseline/foundation was already in a wonky place. I think the next project will hopefully be a little more polished than this one for that reason. I originally started out with a game about a boat in a bowl of cereal, but some of the requirements for the assignment made me change it to something a little more generic with just capturing food items. I think the gameplay loop is pretty basic, but I think that it works overall even if the polish isn't completely there.

GenAI:

Code is one of those interesting things where I actually am not entirely against usage for it since it is just so good at things like debugging. I am typically opposed to AI in any other sense, and I initially started this porject without ever consulting it (other than your GPT models which I appreciated). I think as the project kept growing though, I started having to add things that made me feel kind of incompetent or lost when it came to all of the code that I was using, and I weas finding it hard to focus and really hone in on what I was doing wrong. I started to use AI to debug my code with the help of f12 in the live server and it made it so much easier for me to breakdown and understand. In these instances, I always made sure to see what was wrong as well as WHY it was wrong. Also, I guess it would count as a form of GenAi, but VSCode auto suggestion came in clutch quite a few times when it came to rewriting the code, although it gets it wrong A LOT. All in all, don't really like  AI, but things like debugging make so much sense since it can be time consuming and very confusing when it comes to random typos or missing punctuation.

Others' Code:

I never looked up other people's code because it seemed excessive for a small homework project like this. Maybe I'll take a look if the projects get more intense!
